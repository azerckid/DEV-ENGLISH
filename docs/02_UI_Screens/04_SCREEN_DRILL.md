# Screen Spec: S-03 / S-04 / S-05 드릴 모드
> Created: 2026-03-08
> Last Updated: 2026-03-08

## 1. 화면 목적

게임성 없이 순수 반복 훈련에 집중하는 드릴 모드입니다. S-03(선택 허브) → S-04(빈칸 채우기) 또는 S-05(받아쓰기) 순서로 진입합니다.

---

## 2. S-03 드릴 모드 선택 (`/drill`)

### 2.1 컴포넌트 구성
| 컴포넌트 | shadcn/ui | 설명 |
|---|---|---|
| 빈칸 채우기 카드 | `Card` (클릭 가능) | 클릭 시 S-04로 이동 |
| 받아쓰기 카드 | `Card` (클릭 가능) | 클릭 시 S-05로 이동 |

각 카드 내부:
```
[ 모드 아이콘 ]
[ 모드 이름   ]
[ 한 줄 설명  ]  예: "예문에서 단어를 직접 입력합니다"
```

### 2.2 상태
- 단일 정적 화면 (SSG). 데이터 fetch 없음. 상태 변화 없음.

### 2.3 반응형
| 구간 | 레이아웃 |
|---|---|
| 데스크톱 (≥768px) | 카드 2열 나란히 배치 |
| 모바일 (<768px) | 카드 1열 세로 배치 |

---

## 3. S-04 빈칸 채우기 (`/drill/fill-in`)

### 3.1 화면 목적
예문에서 대상 단어를 빈칸으로 가리고, 사용자가 직접 타이핑해서 맞히는 드릴입니다.

### 3.2 컴포넌트 구성
| 컴포넌트 | shadcn/ui | 설명 |
|---|---|---|
| 진행 상태 표시 | `Progress` + 텍스트 | "3 / 20" + 진행률 바 |
| 예문 표시 영역 | `Card` | 빈칸이 포함된 예문 원문 표시 |
| 빈칸 | - | `_______` 또는 입력 필드 인라인 표시 |
| 단어 입력 필드 | `Input` | 정답 입력. Enter 키로 제출 |
| 제출 버튼 | `Button` | "확인" |
| 피드백 영역 | - | 정답/오답 결과 및 정답 표시 |
| 다음 버튼 | `Button` | 피드백 확인 후 다음 단어로 이동 |
| 출처 표시 | `Badge` (variant: outline) | 예문 출처명 |

### 3.3 상태 정의

| 상태 | 조건 | UI 처리 |
|---|---|---|
| **대기** | 예문 표시, 입력 대기 중 | 입력 필드 포커스 상태 |
| **정답** | 입력값 === 정답 단어 (대소문자 무시) | 피드백 영역: "정답입니다!" (green) + 정답 단어 강조 표시 |
| **오답** | 입력값 !== 정답 단어 | 피드백 영역: "오답입니다" (red) + 정답 단어 즉시 노출 |
| **완료** | 전체 단어 순회 완료 | 결과 요약 화면으로 전환 |
| **Loading** | 단어 목록 fetch 중 | 스켈레톤 표시 |
| **Error** | fetch 실패 | Sonner Toast + 재시도 버튼 |

### 3.4 결과 요약 화면 (완료 시)
| 컴포넌트 | shadcn/ui | 설명 |
|---|---|---|
| 정답 수 / 전체 수 | - | "15 / 20 정답" |
| 다시 시작 버튼 | `Button` | 동일 단어 목록으로 재시작 |
| 홈으로 버튼 | `Button` (variant: outline) | S-01로 이동 |

### 3.5 에러 케이스
| 케이스 | 처리 방식 |
|---|---|
| 단어 목록 없음 | "학습할 단어가 없습니다" + 홈으로 버튼 |
| 빈 입력 제출 | 제출 버튼 비활성화. 입력값 없으면 제출 불가 |

### 3.6 반응형
| 구간 | 레이아웃 |
|---|---|
| 데스크톱 (≥768px) | 중앙 정렬 단일 컬럼, 최대 너비 600px |
| 모바일 (<768px) | 전체 너비, 키보드 올라와도 진행 상태 바 상단 고정 유지 |

## 4. S-05 받아쓰기 (`/drill/dictation`)

### 4.1 화면 목적
단어 발음을 듣고 철자를 직접 타이핑해서 스펠링을 훈련하는 드릴입니다.

### 4.2 컴포넌트 구성
| 컴포넌트 | shadcn/ui | 설명 |
|---|---|---|
| 진행 상태 표시 | `Progress` + 텍스트 | "3 / 20" + 진행률 바 |
| 발음 재생 버튼 | `Button` | Web Speech API `SpeechSynthesisUtterance` 실행 |
| 재생 횟수 표시 | - | "다시 듣기 (N회 남음)" — 힌트 방지용 횟수 제한 옵션 |
| 단어 입력 필드 | `Input` | 철자 입력. Enter 키로 제출 |
| 제출 버튼 | `Button` | "확인" |
| 피드백 영역 | - | 정답/오답 결과 및 정답 표시 |
| 다음 버튼 | `Button` | 피드백 확인 후 다음 단어로 이동 |

### 4.3 상태 정의

| 상태 | 조건 | UI 처리 |
|---|---|---|
| **대기** | 발음 재생 버튼 표시, 입력 대기 | 발음 재생 전까지 입력 필드 비활성화 |
| **재생 중** | TTS 실행 중 | 발음 재생 버튼 로딩 아이콘으로 교체 |
| **입력 대기** | TTS 완료 후 | 입력 필드 활성화 및 포커스 |
| **정답** | 입력값 === 정답 (대소문자 무시) | 피드백: "정답입니다!" (green) |
| **오답** | 입력값 !== 정답 | 피드백: "오답입니다" (red) + 정답 철자 노출 |
| **완료** | 전체 순회 완료 | 결과 요약 화면 |
| **TTS 미지원** | 브라우저가 Web Speech API 미지원 | Sonner Toast: "이 브라우저는 받아쓰기를 지원하지 않습니다" + S-03으로 이동 버튼 |

### 4.4 에러 케이스
| 케이스 | 처리 방식 |
|---|---|
| Web Speech API 미지원 브라우저 | 화면 진입 시 즉시 감지 후 Toast 알림 및 드릴 선택 화면으로 유도 |
| 빈 입력 제출 | 제출 버튼 비활성화 |

### 4.5 반응형
S-04와 동일한 기준 적용.
