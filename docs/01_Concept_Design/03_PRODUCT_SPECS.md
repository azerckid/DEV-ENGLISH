# Product Specs: DEV-ENGLISH
> Created: 2026-03-08 00:22
> Last Updated: 2026-03-08 (§2.5 개인 단어장 추가)

## 1. 개요 (Product Overview)
개발자들이 공식 문서를 쉽게 읽고 이해할 수 있도록 돕는 영어 단어 학습 웹 애플리케이션입니다. 단순한 단어 암기가 아닌 실무 문서(React, Next.js 등)의 예문을 활용하여 맥락적 이해를 돕고, 어원 분석 및 플래시카드 게임을 통해 학습 효율을 높입니다.

## 2. MVP (최소 기능 제품) 정의
MVP는 핵심 가치인 '실전 예문'과 '어원 학습'을 경험할 수 있는 최소한의 범위로 한정합니다.

### 2.1 단어장 (Vocabulary List)
- **실전 예문 연동**: 각 영단어는 실제 프로그래밍 공식 문서에서 크롤링된 예문과 번역을 함께 표시.
- **예문 출처 표기**: 예문이 어느 공식 문서(예: React Docs, MDN)에서 왔는지 명시.

### 2.2 어원 분석 (Etymology Analysis)
- **어근/접두사/접미사 분해 (Morphology)**: 단어를 구성 요소로 쪼개어 보여주고 각 요소의 뜻을 서술.
- **파생어 연결**: 동일한 어원(Root)을 공유하는 다른 개발 용어들을 함께 연결하여 유추 능력을 향상시킴.

### 2.3 학습 모드 (Learning Modes)
학습 목적에 따라 두 가지 하위 모드로 구분합니다.

#### 2.3.1 드릴 모드 (Drill Mode / 연습장)
게임성 없이 순수 학습에 집중하는 반복 훈련 형식입니다.
- **빈칸 채우기 (Fill-in-the-blank)**: 예문에서 단어를 제거하고 사용자가 직접 입력. 오답 시 정답 즉시 노출.
- **받아쓰기 (Dictation)**: 단어 발음을 듣고 철자를 직접 타이핑. 정확한 스펠링 훈련에 집중.
- **진행 방식**: 정답/오답 여부와 무관하게 모든 단어를 순서대로 반복. 게임 점수 없음.

#### 2.3.2 플래시카드 게임 (Flashcard Game / 암기)
학습한 단어를 빠르게 점검하는 게임 형식입니다.
- **카드 뒤집기**: 학습한 단어들을 임의로 섞어 카드 앞/뒷면 뒤집기로 학습(앞: 영어 단어 및 예문 빈칸, 뒤: 뜻과 전체 문장).
- **게임성 (Gamification)**: 맞힘/틀림 버튼을 통한 진행률(Progress) 표시 로직 구현.

### 2.4 텍스트 기반 단어 기여 (Text-to-Vocabulary Contribution)
사용자가 읽던 공식 문서 텍스트를 직접 붙여넣으면, 시스템이 기술 단어를 자동 추출하고 AI로 어원 분석을 생성해 공유 DB에 기여하는 기능입니다.

#### 2.4.1 처리 흐름
1. **텍스트 입력**: 사용자가 공식 문서에서 복사한 텍스트를 입력창에 붙여넣기.
2. **단어 자동 추출**: 시스템이 텍스트에서 기술 용어 후보를 자동 감지 및 추출. 이미 DB에 존재하는 단어는 제외.
3. **추출 결과 확인**: 사용자가 추출된 단어 목록을 확인하고 기여할 단어를 최종 선택.
4. **AI 어원 분석 생성**: 선택한 AI(Claude 또는 OpenAI)로 어원 분해, 형태소 뜻, 파생어를 자동 생성.
5. **DB 기여**: 생성된 데이터가 `unreviewed` 상태로 공유 DB에 저장되어 전체 사용자에게 노출.

#### 2.4.2 AI API 선택
- 사용자가 설정(Settings)에서 Claude API 또는 OpenAI API 중 선택 가능.
- 각자의 API Key를 직접 입력하여 사용 (서비스에서 API 비용 부담 없음).

#### 2.4.3 검수(Review) 정책
- 기여된 단어는 `미검수(unreviewed)` 배지와 함께 즉시 전체 단어장에 노출.
- 운영자 또는 커뮤니티 검수를 통과하면 `검증됨(verified)` 상태로 전환.
- 단어장 필터에서 검수 상태별 조회 가능.

### 2.5 개인 단어장 (My Vocabulary)
로그인한 사용자가 단어를 직접 선택해 저장하거나, 학습 중 틀린 단어가 자동으로 저장되는 개인화 단어장입니다.

#### 2.5.1 단어 추가 방식
- **수동 추가**: S-02 단어 상세 화면에서 "내 단어장에 추가" 버튼 클릭.
- **자동 추가 (오답)**: 다음 두 경우에 해당 단어가 자동으로 개인 단어장에 추가됨.
  - 드릴 모드 (빈칸 채우기 / 받아쓰기): 오답 제출 시
  - 플래시카드 게임: "틀림" 버튼 선택 시
- 이미 개인 단어장에 있는 단어는 중복 추가하지 않음.

#### 2.5.2 개인 단어장 기능
- 저장된 단어 목록 열람 (추가 방식별 필터: 전체 / 수동 / 오답 자동)
- 개인 단어장에서 드릴 모드 / 플래시카드 직접 시작 (내 단어장 기준으로만 학습)
- 단어 개별 삭제

#### 2.5.3 로그인 정책
- 개인 단어장 기능 전체는 로그인 사용자 전용.
- 비로그인 상태에서 "내 단어장에 추가" 또는 오답 자동 추가 시도 시 로그인 유도 Toast 표시.
- 비로그인 사용자의 오답 데이터는 저장하지 않음 (로컬스토리지 임시 저장 없음).

## 3. 플랫폼 요구사항 (Platform Restrictions)
- **반응형 웹 (Responsive Web)**: 데스크톱(코딩 중 띄워놓고 참고) 및 모바일(이동 중 플래시카드 학습) 모두에서 최적화된 UX 제공.
- **고성능 & 오픈소스 (Performance & Open-source)**: GitHub Pages, Vercel 등 정적 웹 호스팅 또는 Edge 네트워크 배포를 기본으로 하며, 빠른 로딩(목표 지연시간 400ms 이하)을 보장.

## 4. 향후 확장 가능성 (Next Steps - Post MVP)
- 크롬 익스텐션(Chrome Extension) 연동: 실제 공식 문서 페이지를 볼 때 모르는 단어를 즉시 번역하고 단어장으로 보내는 기능.
- 커뮤니티 예문 수정(Crowdsourcing): 사용자가 더 정확하고 자연스러운 예문 번역을 PR하거나 제안할 수 있는 기능.
